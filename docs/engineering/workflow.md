# Workflow

## Default Workflow

1. Read the memory system.
2. Read the canonical docs relevant to the task.
3. Inspect the code before changing behavior.
4. Update code.
5. Verify with the smallest meaningful checks.
6. Update docs and operational memory if the change affects them.

## Roadmap Sequencing

- Phase 1: stabilize project surface
- Phase 2: align auth end to end
- Phase 3: build the first real domain slice
- Phase 4: replace placeholder pages with real data
- Phase 5: harden operations and testing

## Documentation Workflow

- Update canonical docs directly.
- Replace duplicate docs with short redirect stubs instead of maintaining parallel narratives.
